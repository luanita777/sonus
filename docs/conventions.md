# Commit Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) standard, where the commit message is structured as follows:  

&nbsp;&nbsp;&nbsp;&nbsp;`<type>[optional scope]: <description>`

Using the following key words:
   
- **feat**: new features  
- **fix**: bug fixes  
- **test**: unit tests  
- **refactor**: code improvements  
- **chore**: setup and configuration  
- **docs**: changes related only to documentation  

### Examples
feat: add song model  
fix: correct ID3 parsing  
test: add DAO tests  
docs: add commit conventions
